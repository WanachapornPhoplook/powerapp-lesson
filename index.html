แนะนำ Power Platform และเริ่มต้นกับ Power Apps

Power Platform

คือชุดเครื่องมือ จาก Microsoft ที่ช่วยให้ทุกคนสร้างโซลูชันทางธุรกิจได้อย่างรวดเร็ว ประกอบด้วยองค์ประกอบหลัก 3 อย่าง

Power Apps

เครื่องมือสำหรับสร้างแอปพลิเคชันทางธุรกิจที่กำหนดเองได้โดยไม่ต้องเขียนโค้ดซับซ้อน

Power Automate

เครื่องมือสำหรับสร้าง Workflow อัตโนมัติเพื่อเชื่อมโยงแอปและบริการต่างๆ เข้าด้วยกัน

Power BI

เครื่องมือสำหรับวิเคราะห์ข้อมูลและสร้าง Dashboard เพื่อแสดงผลข้อมูลเชิงลึก

Power Apps คืออะไร

Power Apps คือแพลตฟอร์ม Low-Code/No-Code สำหรับสร้างแอปพลิเคชันทางธุรกิจได้อย่างรวดเร็วโดยไม่ต้องเขียนโค้ดมากนัก แอปที่สร้างขึ้นสามารถใช้งานได้หลากหลายช่องทาง ไม่ว่าจะเป็นบนมือถือ, เว็บเบราว์เซอร์, หรือฝังใน Microsoft Teams และ SharePoint ได้อย่างง่ายดาย

มีไว้สำหรับใคร?

  • ผู้ใช้ทั่วไปในองค์กร (Citizen Developer): บุคลากรที่อยากสร้างแอปง่ายๆ เพื่อแก้ปัญหาในแผนกของตนเอง
  • นักพัฒนา IT: ใช้สำหรับสร้าง Prototype อย่างรวดเร็วเพื่อทดสอบแนวคิดก่อนการพัฒนาจริง
  • ฝ่ายธุรกิจ/HR/การเงิน: แผนกที่ต้องการสร้างระบบเล็กๆ สำหรับจัดการงานเฉพาะด้านด้วยตนเอง

ใช้งานกับงานอะไร?

  • ฟอร์มเก็บข้อมูล: เช่น ฟอร์มแจ้งซ่อม, ฟอร์มสำรวจความพึงพอใจ
  • แอป Workflow อนุมัติ: เช่น ระบบขอลา, ระบบขออนุมัติจัดซื้อ
  • Dashboard ติดตามงาน: แสดงผลภาพรวมของโปรเจกต์หรือยอดขาย
  • แอปหน้างาน: เช่น แอปเช็คสต็อก, แอปบันทึกการลงเวลางาน

เมนู "Insert" เป็นเหมือนคลังเครื่องมือที่รวบรวมส่วนประกอบ (Controls) ทั้งหมดที่สามารถนำมาใช้สร้างแอปได้ โดยแต่ละ Control จะมีหน้าที่และคุณสมบัติที่แตกต่างกันไป

1. การแสดงข้อมูล (Display)

ใช้สำหรับแสดงข้อมูลหรือข้อความที่ผู้ใช้ไม่สามารถแก้ไขได้

Label

ใช้แสดงข้อความคงที่ เช่น หัวข้อ, คำอธิบาย หรือผลลัพธ์จากการคำนวณ

ยินดีต้อนรับสู่แอป Power Apps

Properties ที่สำคัญ:

  • Text: ข้อความที่จะแสดงใน Label
  • Color: สีของข้อความ
  • Size: ขนาดตัวอักษร
  • Visible: กำหนดว่าจะแสดง Label หรือไม่ (true/false)

HTML Text

แสดงข้อความที่รองรับการจัดรูปแบบแบบ HTML

ข้อความนี้ เป็น HTML

Properties ที่สำคัญ:

  • HtmlText: โค้ด HTML ที่จะแสดง
  • Width / Height: กำหนดขนาดของพื้นที่แสดงผล

 

2. การรับข้อมูลจากผู้ใช้ (Input)

ใช้เพื่อรับข้อมูลที่ผู้ใช้ป้อนเข้าไปในแอป

Text Input

ช่องสำหรับกรอกข้อความทั่วไป

Properties ที่สำคัญ:

  • Default: ค่าเริ่มต้นที่จะแสดงในช่องกรอก
  • HintText: ข้อความแนะนำที่จะแสดงเมื่อช่องว่าง
  • Text: ค่าข้อความปัจจุบันที่ผู้ใช้กรอก

Date Picker

ปฏิทินสำหรับเลือกวันที่

Properties ที่สำคัญ:

  • DefaultDate: วันที่เริ่มต้นที่จะแสดง
  • SelectedDate: วันที่ที่ผู้ใช้เลือกในปัจจุบัน

 

3. การจัดการข้อมูล (Data & Action)

กลุ่ม Control นี้เป็นหัวใจสำคัญที่ทำให้แอปของคุณทำงานได้จริง

Button

ใช้สำหรับกำหนด "การกระทำ" ที่จะเกิดขึ้นเมื่อผู้ใช้กด

Properties ที่สำคัญ:

  • OnSelect: สำคัญที่สุด! เขียนสูตรสำหรับ Action ที่ต้องการ (เช่น `Maps()`, `Collect()`)
  • Text: ข้อความที่จะแสดงบนปุ่ม

Gallery

ใช้แสดงรายการข้อมูลจากแหล่งข้อมูล (เช่น Excel) และปรับแต่งหน้าตาได้

รายการที่ 1
รายการที่ 2
รายการที่ 3

Properties ที่สำคัญ:

  • Items: แหล่งข้อมูลที่จะนำมาแสดง (เช่น ชื่อตาราง Excel)
  • Layout: รูปแบบการแสดงผล (แนวตั้ง/แนวนอน)
  • TemplateSize: ขนาดของแต่ละรายการใน Gallery

 

4. สื่อและอุปกรณ์ (Media & Device)

ใช้สำหรับแสดงรูปภาพ, วิดีโอ หรือเชื่อมต่อกับอุปกรณ์ของเครื่องได้โดยตรง

Camera

ใช้เพื่อเปิดกล้องของอุปกรณ์และถ่ายรูปได้โดยตรงจากในแอป

📸 กล้องถ่ายรูป

Properties ที่สำคัญ:

  • OnStream: การกระทำเมื่อกล้องพร้อมใช้งาน
  • OnPhotoTaken: การกระทำเมื่อถ่ายรูปเสร็จแล้ว

 

หลักการของ Formula

 

1. สัญลักษณ์ทางคณิตศาสตร์ (Arithmetic)

สัญลักษณ์ ชื่อเรียก การทำงาน/การใช้งาน ตัวอย่าง Power Fx
+ บวก (Addition) รวมค่าตัวเลขเข้าด้วยกัน
Revenue_Q1 + Revenue_Q2
- ลบ (Subtraction) หาผลต่างของตัวเลข
TotalStock - ItemsSold
* คูณ (Multiplication) หาผลคูณของตัวเลข
UnitPrice * Quantity
/ หาร (Division) หาผลหารของตัวเลข
TotalScore / MaxScore
% Modulus หารเอาเศษ (หาเศษจากการหาร)
10 % 3

 

2. สัญลักษณ์เปรียบเทียบ (Comparison)

ผลลัพธ์จะเป็น **True** หรือ **False** เสมอ

สัญลักษณ์ ชื่อเรียก การทำงาน/การใช้งาน ตัวอย่าง Power Fx
= เท่ากับ (Equal) ตรวจสอบว่าค่าซ้ายเท่ากับค่าขวาหรือไม่
Status = "Active"
<> ไม่เท่ากับ (Not Equal) ตรวจสอบว่าค่าซ้ายไม่เท่ากับค่าขวาหรือไม่
UserRole <> "Admin"
> มากกว่า (Greater Than) ตรวจสอบว่าค่ามากกว่าหรือไม่
ItemCount > 50
<= น้อยกว่าหรือเท่ากับ ตรวจสอบว่าค่าน้อยกว่าหรือเท่ากับหรือไม่
Cost <= Budget

 

3. สัญลักษณ์ตรรกะ (Logical Operators)

ใช้รวมเงื่อนไขหลาย ๆ เงื่อนไขเข้าด้วยกัน

สัญลักษณ์ ชื่อเรียก การทำงาน/การใช้งาน ตัวอย่าง Power Fx
And / && และ (AND) ต้องเป็น **True ทั้งสองเงื่อนไข** จึงจะได้ True
Filter(Col, Score > 7 && IsActive)
Or / || หรือ (OR) ขอแค่ **เงื่อนไขใดเงื่อนไขหนึ่งเป็นจริง** ก็จะได้ True
Filter(Col, Category = "Book" || Category = "Digital")
Not ไม่ใช่ (NOT) กลับค่าตรรกะ (True เป็น False และกลับกัน)
Not(IsBlank(TextInput1.Text))

 

4. สัญลักษณ์จัดการข้อความและ Record

สัญลักษณ์ ชื่อเรียก การทำงาน/การใช้งาน ตัวอย่าง Power Fx
& การเชื่อม (Concatenation) ใช้เชื่อมข้อความสองส่วนเข้าด้วยกัน
"Invoice-" & Text(InvoiceNumber)
. ตัวดำเนินการจุด (Dot Operator) เข้าถึงคอลัมน์ของ Record หรือคุณสมบัติของ Control
Gallery1.Selected.Name

5. สัญลักษณ์การกำหนดและแบ่งอาร์กิวเมนต์ (Delimiters)

สัญลักษณ์ ชื่อเรียก การทำงาน/การใช้งาน ตัวอย่าง Power Fx
, คอมมา (Comma) ใช้แบ่งอาร์กิวเมนต์/พารามิเตอร์แต่ละตัวในฟังก์ชัน
If(Condition, ResultTrue, ResultFalse)
: โคลอน (Colon) ใช้กำหนดค่าให้กับคอลัมน์ภายใน Record หรือตัวแปร Context (Name: Value)
{ Name: "John", Score: 85 }
{} วงเล็บปีกกา ใช้จัดกลุ่มชุดข้อมูล (Record) หรือตัวแปร Context
Collect(Col, { ID: 1, Name: "Test" })
() วงเล็บ (Parentheses) ใช้ล้อมรอบอาร์กิวเมนต์ทั้งหมดของฟังก์ชันและจัดลำดับการทำงาน
Filter(Col, StartDate = Today())


Formula คืออะไร?

คือชุดคำสั่งที่เราใช้ในการบอก Power Apps ว่าเมื่อเกิดเหตุการณ์บางอย่างขึ้น (เช่น ผู้ใช้กดปุ่ม) แอปจะต้องทำอะไรต่อ



1. IF และ SWITCH (การตัดสินใจตามเงื่อนไข)

ฟังก์ชันสำหรับการตัดสินใจตามเงื่อนไข ใช้เพื่อควบคุมการทำงานของแอปตามตรรกะที่กำหนด

IF

ใช้ตรวจสอบเงื่อนไขแบบ "ใช่/ไม่ใช่" (True/False) เหมาะสำหรับเงื่อนไขที่ไม่ซับซ้อน หรือมีการใช้ AND/OR ร่วมด้วย

SWITCH

ใช้เปรียบเทียบค่าเดียวกับหลายๆ ตัวเลือกที่เป็นไปได้ ทำให้อ่านง่ายกว่าเมื่อมีหลายทางเลือกจากค่าเดียวกัน

ตัวอย่างและการทดลอง: การให้เกรด

ทดลองเปลี่ยนค่าในช่องด้านล่างเพื่อดูผลลัพธ์ของสูตร If(Score >= 80, "A", Score >= 70, "B", "C")

ผลลัพธ์ (เกรด): A

ใช้เพื่อนำผู้ใช้ไปยังหน้าจอ (Screen) อื่นในแอปพลิเคชัน พร้อมทั้งสามารถกำหนดเอฟเฟกต์การเปลี่ยนหน้าได้

คลิกปุ่มเพื่อจำลองการเปลี่ยนหน้าจอด้วยเอฟเฟกต์ Fade

3. COLLECT และ CLEARCOLLECT (การจัดการ Collection)

ใช้สำหรับสร้างและจัดการ "Collection" ซึ่งเป็นตารางข้อมูลชั่วคราวในหน่วยความจำของแอป

Collection: MyCart

4. SEARCH (การกรองข้อมูลตามข้อความ)

ใช้เพื่อกรองข้อมูลในตารางโดยค้นหาข้อความที่ระบุในคอลัมน์ที่กำหนด

ตัวอย่างและการทดลอง: ค้นหาสินค้า

ทดลองค้นหาชื่อสินค้าหรือหมวดหมู่
