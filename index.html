แนะนำ Power Platform และเริ่มต้นกับ Power Apps

Power Platform

คือชุดเครื่องมือ จาก Microsoft ที่ช่วยให้ทุกคนสร้างโซลูชันทางธุรกิจได้อย่างรวดเร็ว ประกอบด้วยองค์ประกอบหลัก 3 อย่าง

Power Apps

เครื่องมือสำหรับสร้างแอปพลิเคชันทางธุรกิจที่กำหนดเองได้โดยไม่ต้องเขียนโค้ดซับซ้อน

Power Automate

เครื่องมือสำหรับสร้าง Workflow อัตโนมัติเพื่อเชื่อมโยงแอปและบริการต่างๆ เข้าด้วยกัน

Power BI

เครื่องมือสำหรับวิเคราะห์ข้อมูลและสร้าง Dashboard เพื่อแสดงผลข้อมูลเชิงลึก

Power Apps คืออะไร

Power Apps คือแพลตฟอร์ม Low-Code/No-Code สำหรับสร้างแอปพลิเคชันทางธุรกิจได้อย่างรวดเร็วโดยไม่ต้องเขียนโค้ดมากนัก แอปที่สร้างขึ้นสามารถใช้งานได้หลากหลายช่องทาง ไม่ว่าจะเป็นบนมือถือ, เว็บเบราว์เซอร์, หรือฝังใน Microsoft Teams และ SharePoint ได้อย่างง่ายดาย

มีไว้สำหรับใคร?

  • ผู้ใช้ทั่วไปในองค์กร (Citizen Developer): บุคลากรที่อยากสร้างแอปง่ายๆ เพื่อแก้ปัญหาในแผนกของตนเอง
  • นักพัฒนา IT: ใช้สำหรับสร้าง Prototype อย่างรวดเร็วเพื่อทดสอบแนวคิดก่อนการพัฒนาจริง
  • ฝ่ายธุรกิจ/HR/การเงิน: แผนกที่ต้องการสร้างระบบเล็กๆ สำหรับจัดการงานเฉพาะด้านด้วยตนเอง

ใช้งานกับงานอะไร?

  • ฟอร์มเก็บข้อมูล: เช่น ฟอร์มแจ้งซ่อม, ฟอร์มสำรวจความพึงพอใจ
  • แอป Workflow อนุมัติ: เช่น ระบบขอลา, ระบบขออนุมัติจัดซื้อ
  • Dashboard ติดตามงาน: แสดงผลภาพรวมของโปรเจกต์หรือยอดขาย
  • แอปหน้างาน: เช่น แอปเช็คสต็อก, แอปบันทึกการลงเวลางาน

ตัวอย่างการใช้งาน Insert Control ใน Power Apps

เมนู "Insert" เป็นเหมือนคลังเครื่องมือที่รวบรวมส่วนประกอบ (Controls) ทั้งหมดที่สามารถนำมาใช้สร้างแอปได้ โดยแต่ละ Control จะมีหน้าที่และคุณสมบัติที่แตกต่างกันไป

1. การแสดงข้อมูล (Display)

ใช้สำหรับแสดงข้อมูลหรือข้อความที่ผู้ใช้ไม่สามารถแก้ไขได้

Label

ใช้แสดงข้อความคงที่ เช่น หัวข้อ, คำอธิบาย หรือผลลัพธ์จากการคำนวณ

ยินดีต้อนรับสู่แอป Power Apps

Properties ที่สำคัญ:

  • Text: ข้อความที่จะแสดงใน Label
  • Color: สีของข้อความ
  • Size: ขนาดตัวอักษร
  • Visible: กำหนดว่าจะแสดง Label หรือไม่ (true/false)

HTML Text

แสดงข้อความที่รองรับการจัดรูปแบบแบบ HTML

ข้อความนี้ เป็น HTML

Properties ที่สำคัญ:

  • HtmlText: โค้ด HTML ที่จะแสดง
  • Width / Height: กำหนดขนาดของพื้นที่แสดงผล

 

2. การรับข้อมูลจากผู้ใช้ (Input)

ใช้เพื่อรับข้อมูลที่ผู้ใช้ป้อนเข้าไปในแอป

Text Input

ช่องสำหรับกรอกข้อความทั่วไป

Properties ที่สำคัญ:

  • Default: ค่าเริ่มต้นที่จะแสดงในช่องกรอก
  • HintText: ข้อความแนะนำที่จะแสดงเมื่อช่องว่าง
  • Text: ค่าข้อความปัจจุบันที่ผู้ใช้กรอก

Date Picker

ปฏิทินสำหรับเลือกวันที่

Properties ที่สำคัญ:

  • DefaultDate: วันที่เริ่มต้นที่จะแสดง
  • SelectedDate: วันที่ที่ผู้ใช้เลือกในปัจจุบัน

 

3. การจัดการข้อมูล (Data & Action)

กลุ่ม Control นี้เป็นหัวใจสำคัญที่ทำให้แอปของคุณทำงานได้จริง

Button

ใช้สำหรับกำหนด "การกระทำ" ที่จะเกิดขึ้นเมื่อผู้ใช้กด

Properties ที่สำคัญ:

  • OnSelect: สำคัญที่สุด! เขียนสูตรสำหรับ Action ที่ต้องการ (เช่น `Maps()`, `Collect()`)
  • Text: ข้อความที่จะแสดงบนปุ่ม

Gallery

ใช้แสดงรายการข้อมูลจากแหล่งข้อมูล (เช่น Excel) และปรับแต่งหน้าตาได้

รายการที่ 1
รายการที่ 2
รายการที่ 3

Properties ที่สำคัญ:

  • Items: แหล่งข้อมูลที่จะนำมาแสดง (เช่น ชื่อตาราง Excel)
  • Layout: รูปแบบการแสดงผล (แนวตั้ง/แนวนอน)
  • TemplateSize: ขนาดของแต่ละรายการใน Gallery

 

4. สื่อและอุปกรณ์ (Media & Device)

ใช้สำหรับแสดงรูปภาพ, วิดีโอ หรือเชื่อมต่อกับอุปกรณ์ของเครื่องได้โดยตรง

Camera

ใช้เพื่อเปิดกล้องของอุปกรณ์และถ่ายรูปได้โดยตรงจากในแอป

📸 กล้องถ่ายรูป

Properties ที่สำคัญ:

  • OnStream: การกระทำเมื่อกล้องพร้อมใช้งาน
  • OnPhotoTaken: การกระทำเมื่อถ่ายรูปเสร็จแล้ว

ภาคปฏิบัติ: สร้างแอปแรกของคุณ

1. หน้าจอเข้าสู่ระบบ (Login Screen) โดย wanachaporn phoplook

 

หลักการของ Formula: หัวใจของการสร้างแอป

สูตรใน Power Apps มีลักษณะคล้ายกับการเขียนสูตรใน Excel แต่ใช้เพื่อกำหนด "การกระทำ" (Action) และ "คุณสมบัติ" (Property) ของส่วนประกอบต่างๆ ในแอป ลองดูตัวอย่างสูตรพื้นฐานที่เราจะใช้ในการสร้างแอป

สูตรพื้นฐานที่ต้องรู้

If() และ Switch()

ใช้สำหรับกำหนดเงื่อนไขในการทำงานของแอป

If(IsBlank(TextInput1.Text), Notify("กรุณาป้อนข้อมูล"), Navigate(Screen2))

Collect(), ClearCollect(), Patch()

ใช้สำหรับเพิ่ม, แก้ไข, ลบข้อมูลใน Collection หรือแหล่งข้อมูล

ClearCollect(colProducts, {ID: 1, Name: "Monitor", Price: 3500})
Patch(colProducts, LookUp(colProducts, ID = 1), {Price: 4000})

Filter(), Search(), LookUp() และ Sort()

ใช้สำหรับจัดการข้อมูลในชุดข้อมูลให้ตรงตามเงื่อนไขที่ต้องการ

Filter(colProducts, Price > 1000)
Sort(Search(colProducts, TextInput1.Text, "Name"), Name, SortOrder.Ascending)

Set() และ UpdateContext()

ใช้สำหรับสร้างและอัปเดตตัวแปรเพื่อเก็บค่าต่างๆ ในแอปพลิเคชัน

Set(varSelectedProduct, ThisItem)


Formula คืออะไร?

คือชุดคำสั่งที่เราใช้ในการบอก Power Apps ว่าเมื่อเกิดเหตุการณ์บางอย่างขึ้น (เช่น ผู้ใช้กดปุ่ม) แอปจะต้องทำอะไรต่อ

ตัวอย่าง: การเปลี่ยนหน้าจอด้วยสูตร Navigate()

สูตรนี้ใช้สำหรับการนำทางจากหน้าจอหนึ่งไปยังอีกหน้าจอหนึ่ง เมื่อคุณเขียนสูตรนี้ใน Property ชื่อ OnSelect ของปุ่ม

ลองดูตัวอย่างสูตร `Navigate()` นี้

Navigate(Screen2, ScreenTransition.Fade)

เมื่อคุณกดปุ่มด้านล่าง แอปจะเปลี่ยนไปหน้าจอที่ 2 โดยใช้เอฟเฟกต์ Fade
